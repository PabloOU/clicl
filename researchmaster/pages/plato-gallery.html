Gallery

Each of the examples below were built both test as well as illustrate Plato's generality and ease-of-use. For each example, you may (1) compile and view the Plato form and (2) edit the declarative description of that form. For those examples drawn from real-world applications, you can also (3) view the original. You may also (4) collaboratively contribute data to the resulting forms (for some of the examples).

For some examples, compilation may take 5-10 seconds, during which the browser's status line will read "Contacting <server>".

  2. On its website, Toyota allows customers to investigate the various options available for each of its cars. This Plato example encapsulates that functionality for the 2009 Toyota Sequoia.   3. Many shopping cart applications ask for shipping and billing addresses, and many offer the ability to automatically populate the billing address with the already-entered shipping address. This example illustrates such functionality.

    Compile | Edit
    This example was developed to illustrate (1) the use of text fields in place of selectors (which at first seem problematic because of their infinite domains) and (2) the automatic population of widget values.

  4. Feature-based programming involves code modules that can be combined only in certain ways. Plato can be used to generate an interface that allows customers to choose among modules and be informed whether or not the selected modules can be combined. More Info

    Compile | Edit
    This version uses checkboxes and radio buttons, ensuring all widgets have values at all times. It illustrates the benefits and drawbacks of using GUI elements with no 'undefined' value.

    Compile | Edit | Collaborate
    This version was designed to enable people to collaboratively make a feature selection. In place of checkboxes and radio buttons, it employs selectors.
