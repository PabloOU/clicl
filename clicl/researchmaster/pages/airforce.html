Introduction

Tag-based authorization is a recent paradigm for controlling which users are granted access to which documents. In tag-based authorization, when a document is contributed to the system, it is labeled with a set of metadata tags describing its content, and when a user is added to the system, she is assigned a set of attributes describing her salient features. The document-release policy for the system is codified by an administrator who writes logical sentences describing the user attributes necessary for accessing documents with particular tags. To address problems with large tag sets, an administrator may also codify the relationships among tags in an ontology.

This site embodies a primitive, web-accessible system demonstrating the key issues in tag-based authorization.

  • Home: This page.
  • Search Docs: Search for documents based on their tags. Results are limited by the authorization policy, though for demonstration purposes you can choose to be whomever you like.
    A simple exercise.
    1. Use the Require Tag drop-down list to select "First Order".
    2. Use the Require Tag list again to select "Reasoning". The displayed documents are tagged with both "First Order" and "Reasoning".
    3. Clear the selected tags using the minus symbols.
    4. Use the Require Tag list to select "Security". Notice there are 0 results.
    5. Change the Login ID to "Timothy L. Hinrichs". Notice there are over 70 results.
  • Add Doc: Add a new tagged document to the system.
  • Authorization Policy: Update the authorization policy. Notice that besides "Timothy L. Hinrichs", several other Login IDs have access to documents tagged with "Security".
  • Tag Ontology: Update how tags are related to one another. For example, a search for "Security" in the Tag interface includes papers not labeled explicitly with "Security" but are rather labeled with "Authorization", "Trust Management", or "Privacy".
