Infomaster Documentation

1. Introduction

Infomaster is an information integration system. It provides integrated access to distributed, heterogeneous information sources, thus giving its users the illusion of a centralized, homogeneous information system. Information consumers can ask questions, confident that the system will provide information from all relevant information sources. Authorized information suppliers can add information updates, confident that the system will distribute those updates appropriately.

An essential feature of Infomaster is its emphasis on structured information sources i.e. databases and knowledge bases. This restriction enables Infomaster to process the information in these sources in a semantic fashion. Information updates can be used to deduce additional updates. Complex queries can be decomposed on semantic grounds into simpler queries, and the answers to these simpler queries can then be combined to form unified answers to the original queries. At various points in this process, information can be translated between different vocabularies and formats. Infomaster includes a world wide web server to support user and administrative interaction with the system via Web browsers like Netscape Navigator and Microsoft Explorer. There is also an electronic interface to allow other programs to query the database contents in query languages like SQL and KIF.

2. Preliminary Details

2.1 Getting the Software

Infomaster is available on the World Wide Web at the URL shown below.

http://logic.stanford.edu/infomaster/

To install the software on your machine, simply copy the application program to an appropriate folder. You may also wish to copy the documentation (i.e. this manual) and some of the sample databases.

Note that there is a substantial charge.

2.2 Configuring Your System

Before you start up Infomaster, you should ensure that your computer's network connections are set correctly. To do this, you need to examine both the Network control panel and the MacTCP control panel.

If you have an Ethernet connection, proceed as follows. Pull down the Apple menu, find the Control Panels menu, and then select the Network option from the corresponding submenu. A dialog box will appear. If the EtherTalk icon is highlighted, everything is fine; otherwise select the EtherTalk icon. In the latter case, you may be asked to confirm the change. Next, pull down the Apple menu again, find the Control Panels menu, and then select the MacTCP option from the corresponding submenu. Again, a dialog box will appear. If the Ethernet icon is highlighted, everything is fine; otherwise select the Ethernet icon. This may result in a message saying that your computer must be restarted for the changes to take effect. If so, you must restart your computer. In any case, note the IP address listed in the dialog box. This is the address to use in communicating with Infomaster in case your machine does not have a symbolic name known to your name server.

If you are running standalone or using Appletalk, proceed as follows. Pull down the Apple menu, find the Control Panels menu, and then select the Network option from the corresponding submenu. A dialog box will appear. If the LocalTalk icon is highlighted, everything is fine; otherwise select the LocalTalk icon. In the latter case, you may be asked to confirm the change. Next, pull down the Apple menu again, find the Control Panels menu, and then select the MacTCP option from the corresponding submenu. Again, a dialog box will appear. If the LocalTalk icon is highlighted, everything is fine; otherwise select the LocalTalk icon. This may result in a message saying that your computer must be restarted for the changes to take effect. If so, you must restart your computer. In any case, note the IP address listed in the dialog box. This is the address to use in communicating with Infomaster.

2.3 Starting Infomaster

You can start Infomaster from the Macintosh Finder by double clicking the program icon. Alternatively, you can select the icon and click Open from the file menu. Once the program begins, you can begin interacting with the system via your Web browser. Note that Infomaster differs from most other application programs in that there is no way of interacting directly with the program; all interaction takes place through your Web browser. Typically, your first step is to load or connect to existing databases. Section 4 describes how to do this. However, it is easier to understand that sort of interaction with the system if you first understand how to use the system once a this has been done. This is described in section 3.

3. Database Interface

3.1 Introduction

Once a copy of Infomaster is running and has accesses to data, you can examine and update the contents of the available databases through the use of Web browsers like Netscape Navigator and Microsoft Explorer.

You can establish a dialog with the system using your browser by linking to your computer at port 4000. For example, to access a copy of Infomaster on a computer named www.epistemics.com, you would type the line shown below. If your machine does not have a name or is not connected to a name server, you should use instead the ip address from the MacTCP control panel.

http://www.epistemics.com:4000

Linking to Infomaster brings up an index page for the database. The index takes the form of a table of classes of objects and operations on those classes. The example shown below illustrates this.

QueryAddModify
CityFind CityAdd CityModify City
RestaurantFind RestaurantAdd RestaurantModify Restaurant

In this example, there are two classes, viz. cities and restaurants; and there are three operations, viz. searching for objects in one of these classes, adding objects, or modifying objects. Clicking on an entry in the table brings up an interaction page appropriate to the corresponding operation and class.

3.2 Simple Queries

Clicking on an entry in the Query column of the Infomaster index page results in a query page specific to the class named by the corresponding row. This query page consists of a form in which you can enter values of attributes of the desired object or objects.

Once you have a query page, you can fill in values for the attributes. This is done in different ways for different attributes. Some attributes offer menus, e.g. city and cuisine in the query page shown above. Some attributes are numerical and allow you to specify upper and/or lower bounds on the attribute's value, e.g. the quality, decor, and price attributes in our restaurant query.

You may select values from menus or fill in values for type-in fields to provide information about the items you are seeking. Multiple selections in a menu are interpreted as alternatives. For example, if you click both San Francisco and Palo Alto, the system assumes you are looking for restaurants in either San Francisco or Palo Alto. You need not make an entry for every attribute. If you do not specify anything, the system assumes you do not care what value or values that attribute has.

Clicking on an attribute name brings up a documentation page for that attribute (provided that the system has any documentation on that attribute. Documentation usually included information on the significance of the attribute, units for physical quantities, and so forth.

Once you have done entering constraints, you can click the Count button to find out how many objects satisfy the values you have specified. In our restaurant example, if we were to click the Count button wih the constraints described above, we might get a result page like the one shown below.

Computing the number of objects is cheaper than displaying those objects with all of their properties. If the number is too large, you might consider adding additional constraints. If the number of solutions is too small, you might want to relax some constraints.

Sometimes, when you have overconstrained a query, you get zero solutions. In this case, Infomaster offers the option of analyzing the query to see where the problem lies. If you click the Analyze button in such situations, the system tries relaxing your constraints one by one and gives you a count of the number of solutions for each case.

4. System Interface

4.1 Introduction

Administration of an Infomaster database is performed through a Web interface similar to that for access to the database itself.

You can query and update system parameters using your browser by linking to port 4002. The following URL is appropriate for a machine named www.epistemics.com and administration port 4002. (Again, if your machine does not have a name or you do not have a name server, then you should use the ip address from the MacTCP control panel.)

http://www.epistemics.com:4002

Opening a connection to this address brings up the index page for system administration. As with the database itself, the index page for system administration contains a table of classes of objects and operations. There are two classes, viz. classes and relations; and there are three operations, viz. searching for objects in one of these classes, adding new objects, or modifying existing objects.

QueryAddModify
ClassFind ClassAdd ClassModify Class
RelationFind RelationAdd RelationModify Relation

Note that, unlike the database query page, there is also a textual interface for free-form interaction with the system. This is used in only a few limited ways (viz. saving and loading databases), and it will be replaced by graphical inputs in future versions of the system.

4.2 Modifying the Interface

Recall that the database index page contains a table of classes and operations. Each of these classes has a set of attributes (so-called relations) that appear on query pages and/or result pages and/or description pages. Each of the attributes that appears on a query page has a particular style (menu or type-in field or numeric range).

You can manage the system's database interface using the administration page in essentially the same manner used in managing the contents of the database itself. For example, you can find out which classes are available by clicking on Find Class. You can modify an existing class by clicking on Modify Class. You can add a new class by clicking on Add Class. Ditto for relations. The following paragraphs describe the various properties of classes and relations.

Each class has three attributes.

  1. The ako attribute tells the system that every instance of a specific class is an instance of the class mentioned in the property. ako stands for a kind of. For example, a bird is a kind of mammal.
  2. The slot attribute relates a class to the relations that are relevant to that class. Some of these relations appear on query pages, some on results pages, and some on description pages, as described below.
  3. The read-only attribute tells the system whether to allow the database user to make additions and modifications to the class.

Each relation has four attributes.

  1. The arity attribute tells the system how many arguments the relation takes. In Infomaster, this is almost always 2.
  2. The style attribute tells the system how to present the attribute on query pages. Possible choice are menu, number, string, typein, and noquery. Menus and numbers are self-evident. The difference between strings and type-in fields is whether or not the characters are to be interpreted verbatim (e.g. case dependent in the case of strings). The noquery option tells the system that the slot is not to be used on the query page (though its values may appear on results or description pages).
  3. The option attribute relates a relation and a possible value for that relation. On query pages, options are presented in the order they are entered into the system.
  4. The documentation attribute gives a documentation string to be used when the database user clicks on that relation name. If no documentation string is supplied, Infomaster does not allow the user to click on the relation name.

Now that you have some sense of the kinds of properties of classes and relations, try some experiments. Examine some classes and relations. Modify existing classes and relations. Try creating new classes with your own relations. Remember that you must go back to port 4000 to see the results of your work at port 4002.

4.3 Saving and Loading Metadata

In order to save or load Infomaster's metadata, click on the ACL icon on the index page. This will bring up a free-form interaction page like the one shown below.

this ACL request

You can use this page to save a database and interface by typing a line into the text entry field of the form (dump-infomaster filename) and pressing the Submit button. The system will return a value indicating the status of the request. If the system returns the filename, this indicates success. If it returns nil, this indicates an error. If this happens, go back and check your typein. Syntax errors make up the majority of problems with this operation.

In order to load a previously dumped metadata, type in a line of the form (load-infomaster filename) and press the Submit button. The system will return a value indicating the status of the request. If the system returns the filename, this indicates success. If it returns nil, this indicates an error. Be sure that your filename refers to a file generated by Infomaster.

Those familiar with KIF (Knowledge Interchange Format) may wish to see and modify the system's metadata in KIF. For these users, there is an additional pair of relevant operations. The system's metadata can be dumped in KIF by typing a line of the form (dump-metadata filename), and metadata can be loaded by typing (load-metadata filename).
